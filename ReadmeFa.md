# 📚 BookTranslatorAI

## مترجم آفلاین کتاب با استفاده از هوش مصنوعی محلی (Local LLM)

**BookTranslatorAI** یک ابزار متن‌باز برای ترجمه کتاب‌های اسکن‌شده است که با استفاده از مدل‌های هوش مصنوعی محلی (Local Large Language Models) و نرم‌افزار Ollama، فرآیند OCR، ترجمه و تولید خروجی را به صورت کاملاً آفلاین انجام می‌دهد.

این پروژه برای افرادی طراحی شده است که می‌خواهند بدون ارسال اطلاعات به سرویس‌های ابری، کتاب‌ها و اسناد اسکن‌شده را با کمک هوش مصنوعی ترجمه کنند.

---


# چرا این پروژه ایجاد شد؟

ایده اصلی این پروژه از یک نیاز ساده شکل گرفت:

> ترجمه یک کتاب اسکن‌شده باید ساده، قابل اعتماد و بدون وابستگی به سرویس‌های ابری باشد.

امروزه بسیاری از ابزارهای ترجمه مبتنی بر هوش مصنوعی به سرویس‌های آنلاین، APIهای تجاری یا زیرساخت‌های پیچیده وابسته هستند. اگرچه این ابزارها امکانات فراوانی دارند، اما برای بسیاری از کاربران که به حفظ حریم خصوصی، اجرای کاملاً آفلاین یا ترجمه مجموعه بزرگی از کتاب‌های اسکن‌شده نیاز دارند، همیشه انتخاب مناسبی نیستند.

BookTranslatorAI با هدف ارائه یک راهکار سبک، شفاف و قابل توسعه ایجاد شد تا با ترکیب OCR، مهندسی پرامپت و مدل‌های زبانی محلی (Local LLM)، فرآیند ترجمه کتاب را تا حد امکان ساده و قابل اعتماد کند.

در طراحی این پروژه تلاش شده است که نرم‌افزار:

- کاملاً آفلاین اجرا شود.
- ساختاری ساده و قابل فهم داشته باشد.
- به‌راحتی قابل شخصی‌سازی باشد.
- دیباگ و توسعه آن آسان باشد.
- روی سخت‌افزارهای معمولی نیز عملکرد مناسبی داشته باشد.

هدف این پروژه رقابت با نرم‌افزارهای حرفه‌ای ترجمه نیست؛ بلکه فراهم کردن یک پایه متن‌باز است تا علاقه‌مندان بتوانند آن را بررسی، توسعه و متناسب با نیازهای خود سفارشی‌سازی کنند.

از هرگونه پیشنهاد، گزارش اشکال، ایده یا مشارکت در توسعه پروژه با کمال میل استقبال می‌شود.

# چرا BookTranslatorAI؟

بیشتر ابزارهای ترجمه مبتنی بر OCR، هر صفحه را مستقل از صفحات دیگر ترجمه می‌کنند.

در این پروژه، تنها **آخرین پاراگراف صفحه قبل** به عنوان زمینه (Context) به مدل داده می‌شود تا:

- پیوستگی متن حفظ شود.
- ترجمه طبیعی‌تر باشد.
- حافظه بسیار کمی مصرف شود.
- سرعت اجرای برنامه بالا باقی بماند.

# مهندسی پرامپت (Prompt Engineering)

در این پروژه به جای استفاده از Translation Memoryهای پیچیده، کیفیت ترجمه با طراحی دقیق پرامپت‌ها افزایش یافته است.

پرامپت‌ها به گونه‌ای طراحی شده‌اند که:

- ساختار متن حفظ شود.
- ترتیب پاراگراف‌ها تغییر نکند.
- از تولید متن‌های تکراری جلوگیری شود.
- صفحات خالی به درستی مدیریت شوند.
- صفحات دارای تنها شماره صفحه باعث Loop نشوند.
- اصطلاحات تخصصی تا حد امکان ثابت باقی بمانند.
- ارتباط صفحات حفظ شود.

تمام پرامپت‌ها در فایل

```
prompts.py
```

قابل ویرایش هستند.
